Debug towards process

发表于2024-10-10|更新于2024-10-20|Debug
|字数总计:359|阅读时长:2分钟|阅读量:

This could be a Tool Supplement of the workflow from Simulink Code Generation to PowerFactory.

PowerFactory (DigISILENT) calls external controller model using MDL model, which is based on dll file. It could be hard if you want to debug the whole process when you use the toolbox to generate the dll. But Visual Studio makes it possible, directly debug towards a process.

But pay attention: By using this debug tool, the dll now must be generated by Visual Studio, rather than our toolbox using gcc. But here we also use our toolbox to generate a clear directory, only delete the generated .dll file. (Also the toolbox can recover some files, which are missing or not correctly generated by Simulink Code Generation! )

The whole process can be divided into 2 steps:

  • Build a dll file using Visual Studio:

    1. create a blank dll project;
    2. right click Header Files in Solution Explorer in the right side menu, add all your relevant .h files in it. You can find these .h files in the path Project_file/inc, which is generated by our toolbox.
    3. similar operation. Add all .c files in Project_file/src in Source files.
    4. navigate to the file IEC61400_template.c, choose Build > Build Solution on the menu bar.
    5. use generated .dll file in PowerFactory.
    6. set breakpoints in the programm.
    7. reference: https://learn.microsoft.com/en-us/cpp/build/walkthrough-creating-and-using-a-dynamic-link-library-cpp?view=msvc-170

  • Monitor PowerFactory:

    1. in Visual Studio, select Debug > Attach to Process (or press Ctrl+Alt+P) to open the Attach to Process dialog box.
    2. set the Connection type to Local or Default.
    3. in the Available processes list, find, and select the process or processes you want to attach to, you can use filter or Track Window to fastly locate the PowerFactory.
    4. in the Code type field, make sure the type of code you plan to debug is listed.
    5. select Attach.
    6. run Simulation or firstly initilize the Simulation in PowerFactory, then it will directly jump to the first breakpoint you make.
    7. reference: https://learn.microsoft.com/en-us/visualstudio/debugger/attach-to-running-processes-with-the-visual-studio-debugger?view=vs-2022
